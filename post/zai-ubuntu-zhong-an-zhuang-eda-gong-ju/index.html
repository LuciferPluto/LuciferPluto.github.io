首先是下载安装包

搭建属于自己的数字IC EDA环境（七）：Ubuntu下ASIC/FPGA环境搭建

根据步骤进行安装，要注意的是：

  1. 要用chmod对文件夹和./synopsysinstaller_v5.0/文件都进行777的重新定义

  2. 在用./setup.sh进行下载时，特别是下载VCS要提前修改系统默认shell

    1. sudo rm /bin/sh
sudo ln -s /bin/bash /bin/sh

  3. verdi下载有问题，经过检查发现是系统磁盘空间不足，不够去加载后续内容

由此，如何在VM中给虚拟机的/home分配额外的空间

首先是先 df -h /home/来检查当前目录下可支配的容量是多少，其次扩展现有分区——需要提前关闭虚拟机

  • 在VMware中，选中已关闭的Ubuntu虚拟机，进入 “虚拟机设置” → “硬盘”。
  • 找到 “扩展” 或 “Utilities” 选项。
  • 输入一个大于当前值的新容量（例如从40GB扩展到100GB），然后点击扩展或确定。

安装分区编辑器：打开终端，输入以下命令安装GParted：

sudo apt-get install gparted

虚拟机下ubuntu home区扩容_ubuntu虚拟机扩容home-CSDN博客

最后别忘了——应用所有操作：

  • 点击GParted工具栏上的 “Apply All Operations”（绿色对勾图标）。系统会开始调整，请耐心等待直至完成。

license生成、环境以及激活

在激活时会遇到如下几个问题：

(13 封私信 / 8 条消息) Ubuntu22.04版本安装VCS+Verdi - 知乎

  1. 在lmgrd命令时，终端会提示找不到lmgrd，没有该文件路径。其在激活时报错如下：

    bash: /home/synopsys/scl/2018.06/linux64/bin/lmgrd: No such file or directory

解决：通过apt下载命令依赖包：

sudo apt-get install lsb-core

  1. 激活的时候报错如下：

    (snpslmd) Can't make directory /usr/tmp/.flexlm, errno: 2(No such file or directory)

    接下来就可以在终端输入“lmg_synopsys”进行激活：

解决：需要在/usr目录下新建立一个tmp文件夹

sudo mkdir -p /usr/tmp
sudo touch /usr/tmp/.flexlm
