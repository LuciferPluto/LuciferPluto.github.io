实习/项目经历

SC_Journey数字模块验证 | 上海南芯科技 | 2025.11-2026.5

项目名称: Journey 多路输出电源管理芯片 (PMIC) — 数字子系统功能验证

Situation (项目背景):
Journey 是一款面向服务器/工业应用的多路输出 PMIC，集成 4路 Buck (最大6A/路，可并联为12A+12A)、4路 LDO、18通道 ADC、I2C/SMBus 接口、WDT 看门狗和 EFUSE 非易失性存储。流片节点为 SMIC 工艺，目标通过功能安全认证。我在实习期间负责数字子系统的功能验证工作。

Task (职责与挑战):
负责数字核心模块的功能验证，包括主状态机 (MAIN_FSM)、I2C/SMBus 通信协议、多层保护机制 (hiccup/latch_off/discharge/deepsafe)、EFUSE/OTP 加载逻辑、WDT 看门狗和 GPIO 控制。挑战在于如何在有限时间内实现高覆盖率收敛，并发现深层次的时序竞态和功能错误。

Action (工作内容):

  • 设计随机化验证框架: 针对保护机制 (8种保护类型 × 3种恢复模式 × 8路输出)，采用 SystemVerilog 约束随机测试，随机配置触发条件、恢复方式和延时参数，单次用例运行 100+ 种子迭代，覆盖率达 99%
  • 验证 I2C/SMBus 协议完整性: 设计单字节/多字节读写、CRC 正确/错误/缺失三种场景、边界地址访问 (0x00/0xFF)、保留寄存器行为测试，发现 2 处 CRC 校验失败后寄存器误更新 bug
  • 开发 EFUSE 全流程验证方案: 构建"烧录 → 断电重启 → 读回比对"闭环测试，对 Main OTP Page 1-4、trim_data、prog_status 分别写入特殊值并验证数据保持性，发现 1 处 Page1 地址映射错误 (base_addr = 0x40 应为 0x80)
  • 定位 EN1 Toggle 竞态条件: 在 PWR_ON 状态下快速 toggle EN1 (<20ms 间隔)，发现状态机卡在 WAIT_EN1_STABLE 无法跳转到 PWR_OFF_SEQ。通过 Verdi waveform 追溯定位为 timer_clear 信号未触发，修复后新增 en1_demo 专项回归
  • 构建 DeepSafe 状态机验证: 验证系统级故障 (I2C CRC 错误、时钟丢失、OTP CRC 错误) 触发 deepsafe 进入/退出流程，检查 DAC=0、ALT 拉低、I2C 命令恢复后 ramp 重建行为，发现 1 处退出时 DAC 直接跳变而非渐变的时序 bug

Result (成果):

  • 交付 25+ 验证用例，功能覆盖率 99%，代码覆盖率 100%，流片前发现 7+ 个功能性 bug (含 2 个状态机竞态、1 个 EFUSE 地址映射、3 个保护读清失效、1 个 DeepSafe 时序)
  • 建立 overnight regression 框架，每次 RTL 变更自动触发 25 用例回归，回归周期 <8 小时
  • 文档化验证策略和 debug 案例，为后续 UVM 迁移提供 reusable 测试组件

LC芯片（Lattice Cryptographic）| SMIC28nm流片 | 南京EDA国创中心 | 2025.3-2025.11

内容：面向低功耗小型终端部署需求的密码学芯片，核心指标782uW@10MHz、版图面积1.563mm×1.807mm，功耗降低22%、面积优化15%

  • 架构设计: 基于 Keccak-f[1600] permutation 设计可配置 SHA-3/SHAKE 加速器，支持 224/256/384/512 位输出，通过 round unrolling (5轮展开) 和 pipeline insertion 实现吞吐提升 40% (vs. 基准架构)，Keccak哈希计算路径并行化优化，SHA-3核心计算效率提升约40%，支持10MHz主频下毫秒级密码学运算
  • 协议设计: 设计低有效双握手 Streaming 接口 (valid/ready + last/tkeep 元数据)，支持变长消息输入 (无需 padding 到固定块大小)，相比传统 AXI-Stream 节省 20% 控制信号开销
  • 验证策略: 采用定向测试 + 随机化组合，设计 NIST Known Answer Test (KAT) 向量比对、随机消息长度/内容交叉校验、边界条件 (空消息/最大长度) 测试，功能覆盖率 99.3% (剩余 0.7% 为 unreachable reserved state)
